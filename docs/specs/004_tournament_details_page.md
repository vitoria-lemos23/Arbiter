- For this task we gonna implement the details page of tournaments. The page will be accessible via the route `/tournaments/:tournamentId` and will display the details of a specific tournament.
- We gonna use the already implemented metadata and indexed tournaments to display the details of the tournament. The page will be a server component that will fetch the tournament details from the metadata store and display them in a user-friendly way.
- For this we gonna need to display the brackets of the tournament, the list of participants, and the details of the tournament such as the name, description, and rules. We will also need to display the status of the tournament (upcoming, ongoing, or completed) and the prize pool if applicable.
- The page will also have a button to join the tournament if the user is eligible to do so. If the user is not eligible, we will display a message indicating why they cannot join the tournament.
- For the brackets, we will use a tree structure to display the matches and the participants. We will also display the results of the matches if they have been completed. The brackets will be displayed in a collapsible format to make it easier for users to navigate through the tournament. 
- We need to reflect about the brackets implementation, since we can use already implemented bracket components or we can implement a new one that fits better with the design of the details page.
- For each match we will support a click event that will open a drawer with the details of the match, including the participants, the score, and the status of the match. The drawer will also have a button to vote for the winner of the match if the user is eligible to do so. If the user is not eligible, we will display a button to see the votes of the match.
- We need to include a edit button for the tournament organizer to edit the tournament details. The edit button will open a modal with a form to edit the tournament details. The form will have fields for the name, description, rules, and other off-chain data. The form will also have a button to save the changes and a button to cancel the changes.
