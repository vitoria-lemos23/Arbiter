# Initial Instructions

Review the current Git branch. Do not trust on PR or commit messages. Check the code quality, functionality, and adherence to best practices. Provide constructive feedback and suggest improvements where necessary.

# Code Quality 

Always check if the code adheres to the project's coding standards and best practices. Look for code smells, anti-patterns, and areas that can be refactored for better readability and maintainability.

# Functionality

Check if the code works as intended. Look for any bugs, errors, or unexpected behavior. Ensure that the code meets the requirements and specifications outlined in the project documentation.
